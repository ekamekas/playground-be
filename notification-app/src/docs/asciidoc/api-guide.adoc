== Introduction
The Notification Service API is a https://aws.amazon.com/what-is/restful-api[RESTful API].
Every endpoints is a resource-oriented APIs, accept and return a http://www.json.org[JSON-encoded] format, and uses standard HTTP status codes.

== Errors

The Notification Service uses HTTP convention status codes to define whether request is success or fail.
`2xx` codes in general are success codes, `4xx` codes are error in information sent (e.g a required parameters are omitted), `5xx` codes are server's errors.

=== HTTP Status

|===
|Code|Description

|200 - OK
|Everything is OK!

|201 - Created
|You have created something, and it is fine

|400 - Bad Request
|Your request is missing something (required parameters maybe)

|401 - Unauthorized
|Valid token or key is not provided

|403 - Forbidden
|You don't have access to resources

|404 - Not Found
|Your particular requested resource is not found

|5xx
|Something wrong with our server, please contact us when it happens :)
|===

=== Types

==== Validation Error

Code: `/errors/validation-error`

Your request may missing one or more required parameters or not fulfilling parameter constraints.

==== Generic Error

Code: `/errors/generic-error`

Fall when not in others error.

=== Response

.Failure
|===
|Path|Type|Description|Optional

|type
|String
|A URI reference [RFC3986] that identifies the problem type
|false

|title
|String
|A short, human-readable summary of the problem type
|false

|detail
|String
|A human-readable explanation specific to this occurrence of the problem
|true

|status
|Number

|false

|instance
|String
|A URI reference that identifies the specific occurrence of the problem
|false

|true
|===
